Technology behind Scenario 1

RHEL 9.8 Image Mode

Immutable edge OS

The edge node runs a cryptographically-verified bootc image. Every service — identity, EDA, Skupper — is baked in. Configuration drift is architecturally impossible. Immutable provenance directly supports RMF/STIG workflows.

MicroShift 4.22

Lightweight Kubernetes at the edge

Runs the jtac-ops namespace with ServiceAccounts, ClusterRoles, and ConfigMaps for all four JTAC operators. RBAC enforcement continues locally even when the OCP API server is unreachable.

Event-Driven Ansible

Autonomous response engine

The EDA rulebook monitors the RHSI health endpoint every 10 seconds. Link loss triggers enforce-local.yml within one poll cycle. Reconnect triggers push-zta-policy.yml. Zero operator action required.

Technology behind Scenario 4

RHEL Image Mode (orbital)

The satellite's immutable OS

phi-4-mini is baked into the bootc image running on PWSA-07. Inference happens on-orbit before downlink. Only verified tracks leave the satellite — conserving precious bandwidth and eliminating cloud dependency.

Red Hat Service Interconnect

Space-to-ground data fabric

Skupper connectors bridge the satellite to four named ground nodes. Each receiver gets the verified track independently over an authenticated RHSI tunnel. The fabric survives partial node failures.

Red Hat ACM

Multi-cluster fleet management

Red Hat Advanced Cluster Management manages the full fleet: the satellite edge cluster plus all ground node clusters. Policy, observability, and zero-trust controls applied uniformly across space-to-ground domain.
